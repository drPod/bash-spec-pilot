The implementation uses libc for POSIX uid/gid/chroot/close-from operations, users for passwd/group database lookups, and tempfile for sudoedit scratch files. It implements common option parsing, command execution, sudoedit basics, user/group switching when permitted, and validation such as rejecting malformed --preserve-env names; full sudoers policy, PAM authentication, plugins, SELinux, ACL/xattr, pty logging, and sparse/network-specific behavior are intentionally skipped.
